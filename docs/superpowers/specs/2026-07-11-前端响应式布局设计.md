# v0.4.0 前端响应式布局生产实现契约

## 1. 文档地位

本文档是 Mutsumi's CARD `v0.4.0` 前端布局的唯一事实来源。其目标不是描述“设计方向”，而是让开发代理即使经过多轮上下文压缩，也能仅凭本文档准确重建 Jetpack Compose 生产界面。

冲突处理顺序：

1. 用户在当前线程中的最新明确要求。
2. 本文档。
3. `docs/superpowers/plans/2026-07-11-v0.4.0全面重构实施计划.md`。
4. 旧规格和 `v0.3.x` 代码。

浏览器原型是视觉参考，不是生产依赖：

`/.superpowers/brainstorm/session-20260706/content/layout-interactive-v3.html`

原型未纳入 Git。本文档必须独立包含实现所需信息，生产代码不得读取或打包原型。

## 2. 不可更改的产品事实

- Android 原生应用：Kotlin + Jetpack Compose + Material 3。
- 当前阶段只提供中文。
- 卡片 `key` 仅为文字；卡片 `value` 仅为一张图片。
- 所有新 value 图片、学习卡片、绘图画布、列表缩略图和详情预览统一为竖向 ISO ID-1 银行卡比例，即 `aspectRatio(53.98f / 85.60f)`。
- value 的逻辑/导出尺寸固定为宽 `1024px`、高 `1624px`。旧 `1024×2048` 图片仅兼容读取并 FitCenter 显示。
- 图片显示使用 `ContentScale.Fit` 或等价 FitCenter；禁止拉伸、裁切、旋转补偿。
- 横屏宽度足够时必须为左、中、右三栏；竖屏必须使用底部导航，即使设备宽度达到 `800dp`。
- 页面不是营销页面。无 hero、无装饰渐变、无嵌套卡片堆叠。
- 学习卡片不使用阴影，实体边界使用 `1dp` 描边。
- 所有操作必须有明确反馈；开发构建不得静默吞异常。

## 3. Compose 生产架构

### 3.1 根组件

生产代码建议按以下边界实现：

```kotlin
@Composable
fun MutsumiCardApp(appContainer: AppContainer)

@Composable
fun AdaptiveScaffold(
    layoutMode: AppLayoutMode,
    selected: AppDestination,
    onSelect: (AppDestination) -> Unit,
    contextPane: (@Composable () -> Unit)?,
    content: @Composable () -> Unit,
)
```

`MutsumiCardApp` 只负责：

- 读取顶层导航状态。
- 装配页面 ViewModel。
- 根据当前页面提供中栏内容和右侧上下文。
- 通过单个 SnackbarHost 展示全局可恢复反馈。

业务数据、搜索、绘图文档和卡片姿态不得直接保存在根 Composable 的普通 `remember` 中。

### 3.2 布局模式

```kotlin
enum class AppLayoutMode {
    Portrait,
    CompactLandscape,
    LandscapeThreePane,
}

fun layoutMode(widthDp: Int, heightDp: Int): AppLayoutMode = when {
    heightDp >= widthDp -> AppLayoutMode.Portrait
    widthDp >= 680 -> AppLayoutMode.LandscapeThreePane
    else -> AppLayoutMode.CompactLandscape
}
```

必须按方向优先判定。以下结果是硬性断言：

| 可用尺寸 | 模式 | 导航 |
| --- | --- | --- |
| `360×800dp` | `Portrait` | 底栏 |
| `800×360dp` | `LandscapeThreePane` | 左栏 |
| `800×1280dp` | `Portrait` | 底栏 |
| `1280×800dp` | `LandscapeThreePane` | 左栏 |
| `640×360dp` | `CompactLandscape` | 底栏 |

`CompactLandscape` 使用单工作区和底栏；不勉强塞入右栏。

### 3.3 状态所有权

必须跨旋转保留：

| 状态 | 所有者 |
| --- | --- |
| 当前主导航目的地 | 根 ViewModel 或 `rememberSaveable` |
| 当前卡组 ID | DataStore + 页面 ViewModel |
| 当前学习卡片 ID、正反面、最近三张 | `StudyViewModel` + SavedStateHandle |
| 卡片搜索词、选中卡片 ID | `CardsViewModel` + SavedStateHandle |
| 录入步骤、key、笔迹、撤销/重做、底图、相机 | `DrawingViewModel` |
| 画笔颜色、笔刷/橡皮大小 | DataStore；ViewModel 持当前值 |
| 导入导出进行中状态 | `BackupViewModel` |

方向变化只能改变组件排列，不能重置上述状态。

## 4. 设计令牌

### 4.1 色彩

| Token | 色值 | 用途 |
| --- | --- | --- |
| `Ink` | `#202623` | 主文字、深色笔刷 |
| `MutedInk` | `#68716C` | 次级文字 |
| `PageBackground` | `#F3F5F2` | 页面背景 |
| `WorkspaceBackground` | `#E7EBE7` | 绘图/卡片舞台 |
| `Surface` | `#FFFFFF` | 内容面、卡面、控件 |
| `Divider` | `#D8DDD9` | 标准分隔线 |
| `StrongDivider` | `#B8C1BB` | 输入框和卡片边界 |
| `PrimaryGreen` | `#285447` | 主按钮、选中状态 |
| `PrimaryGreenSoft` | `#DCE8E2` | 选中背景 |
| `AccentYellow` | `#E2B94F` | 导航指示、轻强调 |
| `DangerCoral` | `#C65F4C` | 忘记、归档、删除 |
| `AssistBlue` | `#496F83` | 辅助图形 |

禁止使用渐变。颜色不能全局变成单一绿色调；红、黄、蓝只用于语义和内容辅助。

### 4.2 形状、描边和阴影

- 标准圆角：`7dp` 或 `8dp`。
- 卡片、输入框、按钮最大圆角：`8dp`。
- 普通分隔线：`1dp Divider`。
- 学习卡片和实体画布：`1dp StrongDivider`；画布可用 `2dp` 强边界强调可绘制范围。
- 学习卡片 `shadowElevation = 0dp`。
- 页面分区使用分隔线，不使用外层悬浮 Card。

### 4.3 间距与触控

- 间距刻度：`4, 8, 12, 16, 24dp`。
- 页面顶栏：`56dp` 高。
- 横屏导航栏：`72dp` 宽。
- 竖屏底部导航：`64dp + WindowInsets.navigationBars`。
- 绘图横屏工具栏：`64dp` 宽。
- 图标按钮视觉尺寸可为 `38-44dp`，语义触控区域必须至少 `44dp`，生产优先 `48dp`。
- 页面主内容不可使用超过 `18dp` 的外边距；学习页竖屏卡片区域横向边距使用 `6-10dp`。

### 4.4 字体

- 页面标题：约 Material `titleLarge`，不做超大展示字。
- 面板标题：`titleMedium`。
- 卡片 key：默认 `30sp`，居中，粗体，固定层级，不随 viewport 宽度线性缩放。
- 条目 key：`14-16sp`、半粗体。
- 次级信息：`10-12sp`。
- 字距必须为 `0sp`，禁止负字距。
- key 容器宽度明确为卡片内部宽度的约 `90%`。中文自然换行，英文长词允许断行。不得收缩成单字符竖列；不得裁切。

## 5. 全局导航壳

### 5.1 目的地

固定四个目的地，顺序不得改变：

1. 学习：`Layers3` 或等价层叠图标。
2. 卡片：`Rows3` 或列表图标。
3. 录入：`PenLine` 或编辑图标。
4. 备份：`Archive` 图标。

设置是横屏导航栏底部的独立入口，不占四个主目的地。竖屏 MVP 可通过备份页或溢出菜单进入设置，不能增加第五个底栏项目。

### 5.2 Portrait / CompactLandscape

组件树：

```text
Scaffold
├─ content: 当前页面（填满扣除 insets 后的空间）
├─ bottomBar: NavigationBar（4 等分目的地）
└─ snackbarHost: 全局 SnackbarHost
```

底栏行为：

- 白色背景，上沿 `1dp Divider`。
- 选中项目使用 `PrimaryGreen` 图标/文字。
- 选中项目顶部有 `28×3dp` 黄色指示线。
- 每项图标在上、中文标签在下，标签约 `10sp`。
- 底栏不得覆盖页面底部操作栏；绘图页底部操作栏位于内容区末端、底栏上方。

### 5.3 LandscapeThreePane

组件树和尺寸：

```text
Row(fillMaxSize)
├─ NavigationRail(width = 72dp)
├─ MainWorkspace(weight = 1f, minWidth = 360dp)
└─ ContextPane(width = clamp(228dp, 25% window width, 310dp))
```

实际右栏宽度公式：

```kotlin
val contextWidth = (windowWidth * 0.25f).coerceIn(228.dp, 310.dp)
```

要求：

- 左栏右侧、右栏左侧均有 `1dp Divider`。
- 左栏顶部为 `38×38dp`、`8dp` 圆角、深绿色背景的品牌标记 “M”。
- 导航项垂直排列，图标上、`9-10sp` 标签下；选中项为浅绿色背景。
- 设置项使用 `Spacer(weight = 1f)` 固定到底部。
- 右栏顶端有 `56dp` 标题行：“上下文 · 当前页面”。
- 右栏内容可纵向滚动；主要提交动作在 `800×360dp` 时仍必须首屏可见。

## 6. 公共页面顶栏

顶栏统一为：

```text
Row(height = 56dp, horizontalPadding = portrait 14dp / landscape 18dp)
├─ Column(weight = 1)
│  ├─ 页面标题（单行、省略）
│  └─ 页面副标题（单行、省略）
└─ Row(actions, gap = 7-8dp)
```

- 白色背景，下沿 `1dp Divider`。
- 图标动作优先，不使用长文字按钮替代熟悉图标。
- 每个图标必须有中文 `contentDescription`；不熟悉图标提供 tooltip。
- 不允许标题或动作改变顶栏高度。

## 7. 学习页

### 7.1 页面层级

```text
StudyScreen
├─ AppTopBar
│  ├─ title = “今日学习”
│  ├─ subtitle = 当前卡组名
│  ├─ 复习方向按钮
│  └─ 上下文按钮（仅窄屏可见）
├─ StudyStage(weight = 1)
│  └─ PhysicalStudyCard(aspectRatio = 53.98f / 85.60f)
└─ FeedbackBar
   ├─ 忘记了（下箭头）
   ├─ 当前序号 / 总数
   └─ 记住了（上箭头）
```

### 7.2 卡片舞台和尺寸

- 舞台背景为 `PageBackground`。
- 卡片始终居中。
- 卡片使用竖向 `aspectRatio(53.98f / 85.60f)`，宽度最多占可用宽度 `80%`。
- 尺寸以可用高度为主要约束，同时不能超过可用宽度：

```kotlin
val cardHeight = min(availableHeight, availableWidth * 2f)
val cardWidth = cardHeight / 2f
```

- Portrait 外边距：顶部/底部由舞台自动分配，左右至少 `6dp`。
- Landscape 卡片不为了填充中栏而变宽，按高度居中。
- 正面白色，`1dp StrongDivider`，`8dp` 圆角，无阴影。
- 正面左上角显示小黄点和当前卡组名；主体只显示 key。
- 背面只显示 value 图片，内边距 `8-10dp`，FitCenter。
- 物理渲染接入后，正反面是同一刚体的固定纹理；渲染层不能切换 Compose 内容树或用 alpha 伪造背面。

### 7.3 反馈栏

- 高度约 `50-54dp`，横向内边距 `10dp`。
- 网格：`1fr / auto / 1fr`，两个按钮等宽。
- 忘记了：白底、珊瑚红文字和边框、下箭头。
- 记住了：白底、绿色文字和边框、上箭头。
- 中间序号使用 tabular number，不因数字位数改变布局。
- 按钮与物理上下滑调用同一 ViewModel action。

### 7.4 横屏上下文栏

顺序固定：

1. 标题“本组进度”。
2. 三个等宽统计格：全部、待复习、已掌握。
3. 标题“接下来”。
4. 最多三项队列：圆形序号、key、权重。

队列不得展示下一张 value，避免泄题。竖屏点击顶栏上下文按钮，使用最高 `72%` 工作区高度的 ModalBottomSheet 展示相同内容。

### 7.5 空状态

空卡组时不显示假卡片或示例图。舞台中心显示“当前卡组还没有卡片”，下方主按钮“录入第一张”，进入录入页。

## 8. 卡片页

### 8.1 页面结构

```text
CardsScreen
├─ AppTopBar
│  ├─ title = “卡片”
│  ├─ subtitle = “卡组名 · N 张”
│  ├─ 新建卡组图标
│  └─ 录入卡片加号
├─ SearchBar(height = 38-40dp)
└─ LazyColumn(CardListItem)
```

- 搜索框和筛选图标在同一行，外边距 `10-16dp`。
- 搜索只匹配 key；MVP 不显示无功能筛选。若筛选尚未实现，应移除筛选按钮，不做假入口。

### 8.2 卡片条目

条目不是 Material Card，而是带底部分隔线的整行：

```text
Row(minHeight = portrait 76dp / landscape 96dp)
├─ ValueThumbnail(aspectRatio = 53.98f / 85.60f, width = portrait 36dp / landscape 42dp)
├─ Column(weight = 1)
│  ├─ key（单行省略）
│  └─ 复习时间、状态、权重
└─ ChevronRight
```

- 条目横向间距 `10-12dp`。
- 选中条目背景 `PrimaryGreenSoft` 的低透明版本，左侧 `3dp PrimaryGreen` 指示线。
- 选中状态不能改变条目尺寸。
- 丢失图片的条目显示明确错误图标和“图片缺失”，不能崩溃整个列表；开发日志保留异常。

### 8.3 详情映射

Landscape 右栏：

1. 当前 key。
2. 居中的竖向预览，宽度不超过 `150dp`，采用银行卡比例。
3. 复习状态与权重。
4. 并排“编辑”“归档”按钮。
5. 删除位于确认对话框后的危险操作，不与归档混淆。

Portrait / CompactLandscape：点击条目打开 ModalBottomSheet，内容和操作一致，最大高度约 `72%`。编辑进入录入编辑流程；重新绘制加载当前 value 为底图或文档底图。

### 8.4 卡组交互

- 顶栏显示当前卡组。
- 支持创建、改名、切换卡组；使用菜单或底部弹层，不额外增加页面卡片。
- 新安装仅创建空“默认卡组”，禁止注入示例卡片。
- 空卡组显示“录入第一张”。
- 搜索无结果保留搜索框，显示“没有匹配的 key”和清除搜索图标。

## 9. 录入页

### 9.1 竖屏步骤一：填写 key

组件树：

```text
DrawScreen(step = Key)
├─ AppTopBar(title = “录入卡片”, subtitle = “先填写文字 key”)
└─ ScrollableColumn(maxWidth = 560dp, horizontalPadding = 16dp)
   ├─ “步骤 1 / 2”
   ├─ 标题“这张卡片要记什么？”
   ├─ 副文案
   ├─ OutlinedTextField(label = “文字 key”)
   ├─ 三条规则
   └─ PrimaryButton(fillMaxWidth, text = “进入绘图”)
```

规则必须准确写明：

1. value 只保存图片，新画布/成品为竖向 `1024×1624`。
2. 底图 FitCenter，不拉伸；预览与导出一致。
3. 触控笔绘制，双指移动和缩放逻辑画布。

key 为空时：字段 `isError=true`，显示字段错误“请输入文字 key”，同时可发 Snackbar；不进入步骤二。

### 9.2 竖屏步骤二：绘图

```text
Column
├─ AppTopBar(title = “录入卡片”, subtitle = 当前 key, back = 返回 key)
├─ ToolArea(height ≈ 88dp)
│  ├─ Row: 画笔 / 橡皮 / 移动 / Spacer / 底图
│  └─ Row: 4 色 swatch / Spacer / 笔刷圆点 / 水平 Slider
├─ CanvasStage(weight = 1, background = WorkspaceBackground)
│  └─ DrawingCanvas(aspectRatio = 53.98f / 85.60f, fit = height and width)
└─ BottomActions(height ≈ 54dp)
   ├─ 撤销图标（44dp）
   ├─ 重做图标（44dp）
   └─ 保存卡片（weight = 1）
```

核心工具不得放入横向滚动容器。颜色与笔刷大小必须一直可见。

### 9.3 横屏绘图

外层三栏已提供全局左导航和右上下文。中栏内部：

```text
Row
├─ DrawingToolRail(width = 64dp)
│  ├─ 画笔
│  ├─ 橡皮
│  ├─ 移动
│  ├─ 2×2 色点
│  ├─ 竖向 Slider
│  └─ 底图
└─ CanvasStage(weight = 1)
   └─ DrawingCanvas(aspectRatio = 53.98f / 85.60f)
```

右上下文栏顺序：

1. 标题“卡片信息”。
2. key 输入。
3. 标题“图片预览 · 竖向 2:1”。
4. 竖向预览。
5. 保存和清空。

当可用高度 `<=440dp`：预览宽度缩至约 `54dp`，保存/清空改为并排，二者必须首屏完整可见。

### 9.4 画布和相机

- 逻辑边界：`Rect(0, 0, 1024, 1624)`。
- 初始相机显示约逻辑画布的 `35%`，但视觉上的实体画布始终完整有边界；相机用于在逻辑空间中移动/缩放内容，不得在手势结束时弹回初始值。
- 相机最小/最大缩放由能覆盖画布和 `6x` 上限决定。
- 所有绘制点先转换到逻辑坐标，再裁剪到实体画布边界。
- 画布外区域为 `WorkspaceBackground`，不可写入。
- 底图 FitCenter 到 `1024×1624`，空白区域为白色。
- Compose 预览和 PNG 导出调用同一 `DrawingSceneRenderer` 几何。
- 导出固定 `1024×1624 PNG`、白色背景。
- 单点笔触渲染为圆点。
- 仅底图允许保存；底图和笔迹都为空则禁止保存。

### 9.5 输入行为

- 触控笔/单指：当前绘制工具。
- `PointerType.Eraser`：临时橡皮，不永久改变工具选择。
- 双指：移动/缩放。
- 移动工具下单指可平移。
- `pointerInput` 不得以每帧变化的 Camera 值作为 key；使用稳定状态对象或 `rememberUpdatedState`，避免手势被重启。

## 11. 反馈、错误和加载

- 全局使用单一 SnackbarHost，页面级字段错误仍贴近字段。
- 成功反馈：保存卡片、导出完成、导入完成、卡组创建/改名。
- 取消反馈：SAF 导入/导出取消。
- 确认对话框：清空画布、归档、永久删除、导入覆盖风险。
- 加载期间保持布局尺寸稳定；用 ProgressIndicator 替换按钮内部图标/文本，不改变按钮大小。
- 开发构建：未知异常记录完整堆栈并重新抛出，禁止空 `catch`。
- 发布构建：用户看到中文摘要，日志保留原始异常和上下文。

## 13. 四尺寸生产验收

### 13.1 `360×800dp`

- 底部导航存在，左导航和右栏不存在。
- 学习竖卡占据主视觉区域，反馈栏和底栏均完整可见。
- 录入步骤一按钮可见；步骤二两行工具、画布、底部操作均可达。
- 画布不横向溢出。

### 13.2 `800×360dp`

- 三栏存在：`72dp` 左栏、至少 `360dp` 中栏、`228-310dp` 右栏。
- 学习 key 完全位于竖卡内，不形成单字符溢出列。
- 录入中栏存在 `64dp` 工具栏，竖画布完整居中。
- 右栏 key、缩小预览、保存、清空首屏可见。

### 13.3 `800×1280dp`

- 必须仍为底部导航，不能因宽度 `800dp` 误切三栏。
- 学习卡片按高度和宽度共同约束，不超出舞台。
- 卡片详情使用底部弹层。

### 13.4 `1280×800dp`

- 三栏比例稳定，右栏不超过 `310dp`。
- 卡片列表为条目，不变成网格。
- 详情预览为竖向银行卡比例，操作按钮不被推出首屏。

### 13.5 所有尺寸共同断言

- 页面根节点 `scrollWidth == clientWidth` 的等价 Compose 条件：无横向越界布局。
- 导航不遮挡内容。
- 所有 value 表面测得 `width / height == 0.5`（允许像素取整误差）。
- 图片没有拉伸或裁切。
- 文本不与前后元素重叠。
- 切换四个目的地可用。
- 旋转前后导航、选择、录入草稿和学习卡片保持。
- 无假按钮、无静默点击。

## 14. 禁止实现清单

- 禁止把 value、画布或卡片恢复为横向 `2:1`。
- 禁止仅按宽度断点让竖屏平板进入三栏。
- 禁止横屏只有导航栏和单内容区而缺少上下文栏。
- 禁止列表使用嵌套 Card 或卡片网格。
- 禁止学习卡片阴影和透明虚影。
- 禁止用 alpha 切换两张 Compose 卡面来实现物理翻转。
- 禁止工具栏横向滚动隐藏颜色、笔刷大小、底图或保存。
- 禁止底图 `FillBounds` 或裁切。
- 禁止注入示例卡片。
- 禁止无反馈的保存、导入、导出、清空、归档和删除。
- 禁止为了让测试通过而缩减已确认页面或交互。
