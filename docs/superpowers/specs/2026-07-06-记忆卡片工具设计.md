# 记忆卡片工具设计

## 背景

本项目是一个安卓端记忆卡片工具。卡片的 `key` 仅为文字，`value` 仅为图片。第一版目标是提供简洁的复习展示、轻量随机推荐、卡组管理、触控笔图片录入，以及本地备份能力。

所有项目文档、规格说明、提交信息和应用内中文文案默认使用中文。代码标识符沿用 Android 与 Kotlin 生态的英文命名习惯。

## 第一版范围

第一版采用轻量原生 MVP 路线：

- 原生 Android 应用。
- 使用 `Kotlin + Jetpack Compose + Material 3`。
- 使用 `Room` 保存结构化数据。
- 使用应用私有目录保存图片资源。
- 使用 `DataStore` 保存用户设置。
- 支持手动导出和导入备份包。
- 支持横屏三栏和竖屏底部导航的响应式布局。
- 支持轻量推荐权重，不实现完整 Anki 式间隔重复。

第一版不做账号体系、云同步、多标签、复杂多图层绘图、完整间隔重复算法。

## 产品结构

应用主导航包含四个目的地：

- 学习：默认首页，用于随机推荐和复习浮动卡片。
- 卡片：按卡组展示卡片条目，支持查看、搜索和编辑入口。
- 绘制/新增：输入文字 `key`，导入底图，使用触控笔绘制或标注，保存为图片 `value`。
- 备份：手动导出和导入备份包。

学习页默认以 `key -> value` 的方向复习：先显示文字，用户点击或手势操作后显示图片。应用也支持反向练习，即先显示图片，再回忆文字。

复习反馈使用三个按钮：

- 记不住：提高后续推荐权重。
- 模糊：轻微提高后续推荐权重。
- 记住了：降低后续推荐权重。

## 响应式布局

布局以学习页为核心。

横屏或大屏使用三栏：

- 左栏：导航 rail，包含学习、卡片、绘制/新增、备份。
- 中栏：主要复习区域，使用浮动卡片展示当前推荐卡片。
- 右栏：推荐队列、当前卡组概览和卡片详情。

竖屏或窄屏使用底部导航：

- 左侧导航 rail 移到底部导航栏。
- 中央保留浮动卡片作为主要内容。
- 右栏内容折叠为底部弹层或二级页面。

建议断点：

- 宽度大于等于 `840dp` 时使用三栏布局。
- 宽度小于 `840dp` 时使用底部导航布局。

## 技术架构

应用采用单 Activity 架构：

- `MainActivity` 承载 Compose 根界面。
- Compose Navigation 管理页面切换。
- `ViewModel + Kotlin Flow` 驱动页面状态。
- Repository 封装 Room、文件资源和备份读写。

主要模块：

- `study`：推荐卡片、浮动卡片状态、复习方向、反馈处理。
- `cards`：卡组列表、卡片条目、搜索、预览和编辑入口。
- `draw`：key 输入、底图导入、触控笔画布、图片保存。
- `backup`：导出、导入、冲突处理和结果提示。
- `data`：Room 实体、DAO、Repository 和备份序列化。
- `ui/adaptive`：横屏三栏和竖屏底部导航布局壳。

## 数据模型

第一版使用以下核心实体。

`DeckEntity` 表示卡组：

- `id`：卡组 ID。
- `name`：卡组名称。
- `createdAt`：创建时间。
- `updatedAt`：更新时间。

`CardEntity` 表示卡片：

- `id`：卡片 ID。
- `deckId`：所属卡组 ID。
- `keyText`：文字 key。
- `valueImagePath`：图片 value 的应用私有目录相对路径。
- `baseImagePath`：可选底图相对路径。
- `createdAt`：创建时间。
- `updatedAt`：更新时间。
- `archived`：是否归档。

`ReviewStateEntity` 表示推荐状态：

- `cardId`：卡片 ID。
- `weight`：推荐权重。
- `seenCount`：出现次数。
- `againCount`：记不住次数。
- `unsureCount`：模糊次数。
- `knownCount`：记住次数。
- `lastReviewedAt`：最后复习时间。

`AppSettings` 使用 DataStore 保存：

- 当前卡组 ID。
- 当前复习方向。
- 最近使用画笔颜色。
- 最近使用画笔粗细。
- 最近使用橡皮粗细。

## 推荐权重

随机推荐以当前卡组为范围。第一版不计算复杂记忆曲线，只维护每张卡片的推荐权重。

初始权重为 `1.0`。用户反馈后更新：

- 记不住：`weight = min(weight + 0.8, 5.0)`。
- 模糊：`weight = min(weight + 0.3, 5.0)`。
- 记住了：`weight = max(weight * 0.55, 0.2)`。

抽卡时使用加权随机。为避免连续重复，最近出现的 3 张卡片临时降低权重。若当前卡组卡片数量不足，则允许重复。

推荐算法封装为独立接口，后续可替换为完整间隔重复策略。

## 触控笔录入

绘制/新增页包含：

- key 文本输入。
- 底图导入。
- 画笔颜色。
- 画笔粗细。
- 橡皮。
- 撤销。
- 重做。
- 清空。
- 保存。

画布支持触控笔和手指输入。底图作为背景层渲染，用户笔迹作为绘制层。保存时将底图和笔迹合成为一张图片，作为卡片的 `value`。

第一版不做多图层、套索、选择移动和手写识别。

## 备份格式

导出文件为 ZIP 包，包含：

- `manifest.json`：备份版本、导出时间、应用版本、卡组数量、卡片数量。
- `database.json`：卡组、卡片、复习状态的结构化快照。
- `images/`：value 图片和底图资源。

导入时读取 `manifest.json` 校验版本，再导入 `database.json` 和图片资源。若检测到 ID 冲突，第一版采用“导入为新副本”的策略：生成新的卡组 ID 和卡片 ID，避免覆盖现有数据。

## 错误处理

应用需要处理以下情况：

- 图片导入失败：提示用户重新选择图片。
- 图片保存失败：提示保存失败，并保留当前画布状态。
- 卡组为空：学习页显示创建卡片入口。
- 当前卡片图片丢失：卡片条目标记异常，并允许重新编辑保存。
- 备份包版本不兼容：停止导入并显示原因。
- 导入过程中部分资源失败：导入完成后显示失败数量和可恢复提示。

## 测试策略

第一版测试重点：

- 推荐权重更新和加权随机的单元测试。
- Room DAO 的读写测试。
- 备份导出和导入的集成测试。
- 横竖屏布局的 Compose UI 测试。
- 画布保存结果的基础测试，确认输出图片非空且路径可读取。

## 验收标准

第一版完成时应满足：

- 用户可以创建卡组。
- 用户可以创建文字 key 和图片 value 的卡片。
- 用户可以导入底图并使用触控笔标注保存。
- 用户可以在学习页按当前卡组随机复习。
- 用户反馈后，推荐权重会变化。
- 横屏显示左中右三栏。
- 竖屏显示底部导航。
- 用户可以导出和导入备份包。
