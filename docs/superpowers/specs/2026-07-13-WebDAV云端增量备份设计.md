# WebDAV 云端增量备份设计

## 目标与范围

在现有备份页内增加手动云端增量备份，服务端使用 AList 提供的标准 WebDAV。功能不包含后台自动任务、弹窗、多设备双向合并和冲突解决。

云端恢复沿用本地备份的安全语义：校验后导入为新的本地副本，不覆盖或删除当前数据。

## 页面布局

- 页面本身允许纵向滚动，不使用对话框、底部弹层或独立设置页。
- 竖屏依次显示云端备份、本地备份和最近记录。
- 横屏将云端备份放在左侧主区域，本地备份与最近记录放在右侧。
- 云端版本使用横向滑动窗口，最多展示最近 8 个版本。
- “本次变化”只显示 Git 风格统计：`+N` 为新增或内容发生变化的卡片，`-N` 为删除的卡片，不展示完整文件列表。
- 未配置时在云端区域内直接展示 WebDAV 地址、用户名、密码和远端目录输入框。已配置时默认折叠，设置按钮在原位置展开。

## 远端格式

远端根目录默认为 `MutsumiCard`：

```text
MutsumiCard/
  index.json
  snapshots/<timestamp>-<uuid>.json
  objects/<sha256>.png
```

`index.json` 保存格式版本和最近 8 个版本摘要。每个摘要包含版本 ID、创建时间、`+N/-N`、卡片数量和本版本引用的图片哈希。

快照 JSON 保存完整的卡组、卡片、复习状态，以及本地图片路径到 SHA-256 对象名的映射。图片对象以内容哈希命名，相同 PNG 只上传一次。

## 备份流程

1. 校验 HTTPS 地址、用户名、密码和安全远端路径。
2. 读取 `index.json`；不存在时视为空仓库，其他 HTTP 错误必须显示。
3. 读取最新快照并按卡片 ID 比较指纹。指纹仅包含 key、卡组、归档状态和图片 SHA-256；权重、复习次数、反馈计数和最后复习时间不参与增量统计，但仍完整写入快照并可恢复。
4. 计算 `+N/-N`。新增和指纹变化计入 `+N`，远端存在而本地不存在计入 `-N`。
5. 创建远端目录，只上传索引中尚未引用的图片对象。
6. 上传新快照，最后上传新索引。索引更新前失败不得影响旧版本。
7. 删除滑动窗口外的快照和不再被保留版本引用的图片。清理失败作为明确警告返回，不静默处理。

## 恢复流程

1. 用户在横向版本窗口点击恢复。
2. 下载并解析对应快照，校验数据关系和所有路径。
3. 下载引用图片并逐个验证 SHA-256。
4. 复用现有 Repository 导入逻辑创建新的卡组和卡片副本。
5. 临时文件必须清理；清理失败显示警告并进入现有清理诊断路径。

## 网络与配置

- 使用 OkHttp 执行 `GET`、`PUT`、`DELETE` 和 `MKCOL`，仅依赖标准 WebDAV，不调用 AList 私有 API。
- 使用 HTTP Basic Authentication；只接受 `https://` 服务地址。
- 配置保存在应用私有存储中。密码不会写入项目备份 ZIP 或云端快照。
- 所有网络错误、认证失败、格式错误和清理失败都必须形成中文反馈；未知程序异常继续抛出。

## MVP 限制

- 假设同一远端目录仅由一个设备写入，不实现并发写入保护。
- 不加密云端快照内容，安全性依赖 HTTPS、AList 权限和存储端访问控制。
- 不自动定时备份，用户必须在备份页点击“立即增量备份”。
