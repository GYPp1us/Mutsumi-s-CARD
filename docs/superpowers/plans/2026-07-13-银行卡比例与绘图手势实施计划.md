# 银行卡比例与绘图手势实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 修复学习页留白、统一银行卡比例，并让绘图跨旋转持久且移动缩放连续跟手。

**Architecture:** 以单一画布规格驱动渲染和校验；以 `DrawingViewModel` 持有业务状态；以稳定手势协程和局部累计相机处理 transform。

**Tech Stack:** Kotlin、Jetpack Compose、ViewModel、Android Bitmap。

---

### Task 1: 统一比例

- [ ] 将逻辑和导出画布改为 `1024×1624`、比例 `0.6306075f`。
- [ ] 更新 PNG 校验与所有卡片 `aspectRatio` 调用。
- [ ] 学习页宽度上限设为可用宽度 `80%`。

### Task 2: 持久绘图状态

- [ ] 让生产 `DrawScreen` 使用 `DrawingViewModel` 的 key、步骤、文档、工具和相机。
- [ ] 底图只保存字节，由渲染层按需解码并释放。
- [ ] 旋转仅调用相机 `withViewport`。

### Task 3: 连续变换手势

- [ ] 抽取以实时质心为锚点的相机变换函数。
- [ ] `pointerInput` 使用稳定 key，单次手势累计最新相机。
- [ ] 横竖屏画布均以宽高双约束居中。

### Task 4: 远程候选版

- [ ] 执行静态差异检查与远程 APK 编译。
- [ ] 发布签名 `v0.4.1`，由用户真机验证旋转和双指手势。
