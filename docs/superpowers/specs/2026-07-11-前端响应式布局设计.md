# 前端响应式布局设计

## 文档状态

本文档记录阶段二、阶段三已经通过交互原型确认的前端布局。它是后续 Jetpack Compose 实现和 UI 验收的依据。

本文档只约束界面结构、响应式行为、视觉层级和录入流程，不定义数据库迁移、推荐算法、卡片物理手势引擎、完整备份实现或生产发布流程。

交互原型位于：

`/.superpowers/brainstorm/session-20260706/content/layout-interactive-v3.html`

## 设计目标

- 学习页保持安静、聚焦，实体卡片始终是页面主角。
- 卡片正面只显示文字 key，背面只显示图片 value。
- value 图片和实体卡片统一为竖向 2:1，逻辑分辨率固定为宽 `1024`、高 `2048`。
- 横屏提供左、中、右三栏，充分利用平板空间。
- 竖屏使用底部导航和单工作区，避免横屏结构被压缩后直接复用。
- 录入流程适合触控笔，核心工具始终可见，不依赖横向滚动寻找。
- 所有操作必须有明确结果反馈；开发构建不得静默吞掉异常。

## 视觉基线

### 视觉语言

界面采用安静、工具化的视觉风格：

- 页面背景使用浅灰绿色，内容面使用白色。
- 深绿色用于主操作和当前状态。
- 黄色用于导航指示和少量强调。
- 珊瑚红用于遗忘、归档等负向操作。
- 蓝灰色用于辅助图形和次级信息。
- 不使用渐变、装饰光斑或大面积单色主题。
- 页面区块不包装成悬浮卡片；卡片样式只用于真实卡片、重复条目和弹层。
- 卡片圆角和控件圆角不超过 `8dp`。
- 学习卡片不使用阴影，只使用 `1dp` 边框建立实体边界。

### 基础颜色

| 用途 | 色值 |
| --- | --- |
| 主文字 | `#202623` |
| 次级文字 | `#68716C` |
| 页面背景 | `#F3F5F2` |
| 工作区背景 | `#E7EBE7` |
| 内容面 | `#FFFFFF` |
| 分隔线 | `#D8DDD9` |
| 主绿色 | `#285447` |
| 强调黄色 | `#E2B94F` |
| 警示珊瑚红 | `#C65F4C` |
| 辅助蓝灰色 | `#496F83` |

### 间距与尺寸

- 基础间距使用 `4dp`、`8dp`、`12dp`、`16dp`、`24dp`。
- 页面顶栏标准高度为 `56dp`。
- 竖屏底部导航标准高度为 `64dp`，另加系统安全区。
- 横屏导航栏标准宽度为 `72dp`。
- 可点击控件的有效触控范围不小于 `44dp`，生产实现优先达到 `48dp`。
- 图像、画布和固定格式控件必须通过明确的宽高比约束尺寸，动态内容不得推动整体布局跳动。

## 响应式框架

### 模式判定

布局模式以方向优先，不单纯以宽度决定：

| 条件 | 布局模式 |
| --- | --- |
| 竖屏，包括 `800×1280` 平板 | 单工作区加底部导航 |
| 横屏且可用宽度不小于 `680dp` | 导航栏、主工作区、上下文栏三栏 |
| 横屏但可用宽度小于 `680dp` | 紧凑单工作区加底部导航 |

旋转时只重排现有状态，不重建业务流程。当前导航目的地、当前卡片、选中条目、key 输入、绘图步骤和画布状态都必须保留。

### 竖屏壳

竖屏根布局由两部分组成：

1. 上方单一工作区，承载当前页面。
2. 下方四项底部导航，依次为学习、卡片、录入、备份。

横屏右栏中的信息在竖屏通过以下方式承载：

- 临时信息和快速操作使用底部弹层。
- 需要持续编辑的内容进入二级页面。
- 弹层高度最多占可用工作区的约 `72%`，点击遮罩可关闭。

### 横屏壳

横屏三栏结构固定为：

| 区域 | 尺寸 | 职责 |
| --- | --- | --- |
| 左栏 | `72dp` | 主导航和设置入口 |
| 中栏 | `minmax(360dp, 1fr)` | 当前任务的主要工作区 |
| 右栏 | `228dp` 至 `310dp` | 当前页面的上下文、预览和操作 |

右栏不能承担页面主任务，也不能侵占中栏的最小工作宽度。右栏内容允许纵向滚动，但主要提交操作在常见横屏高度中应保持首屏可达。

## 主导航

主导航包含四个稳定目的地：

- 学习：随机推荐和复习。
- 卡片：卡片浏览、搜索和管理。
- 录入：填写文字 key，并绘制图片 value。
- 备份：本地 ZIP 导入和导出。

竖屏和横屏使用同一组目的地与选中状态，只改变导航控件的位置和形态。

## 学习页

### 页面结构

学习页按从上到下的顺序包含：

- 顶栏：页面标题、当前卡组、复习方向和上下文入口。
- 卡片舞台：承载一个物理一体化学习卡片。
- 反馈区：忘记了、当前序号、记住了。

反馈按钮是手势的可访问性替代入口。阶段四接入物理手势后，按钮与上滑、下滑必须调用同一业务动作。

### 卡片尺寸

- 卡片统一为竖向 2:1，即宽高比 `1:2`。
- 卡片正反面尺寸完全相同。
- 正面只显示文字 key，背面只显示图片 value。
- 背面图片使用 `FitCenter`，禁止拉伸和裁切。
- 竖屏下卡片优先占满可用高度，同时受工作区宽度限制。
- 横屏下卡片按可用高度居中，不为了填满中栏而改变比例。
- 卡片周围只保留必要操作空间，不添加宽边框、大外边距或阴影。

### 文字适配

- key 默认使用固定层级的标题字号，不随屏幕宽度无限放大。
- 中文可按自然字符换行，英文长词允许安全断行。
- 文本容器使用明确宽度，不能收缩为单字符列。
- 最长 key 必须完整位于卡面内；必要时减少字号或增加行数，不能裁切。

### 横屏右栏

学习页右栏显示：

- 当前卡组总数、待复习数和已掌握数。
- 接下来若干张卡片的 key 和权重摘要。

右栏只显示队列摘要，不提前渲染下一张卡片的 value。

## 卡片页

### 竖屏

- 顶栏显示当前卡组和卡片数量。
- 顶栏提供新建卡组和录入卡片入口。
- 搜索框固定在条目列表上方。
- 卡片使用单行条目，不使用卡片网格。
- 条目包含竖向 `1:2` 缩略图、key、复习摘要和详情指示。
- 点击条目后以底部弹层显示预览和操作。

### 横屏

- 中栏显示搜索和卡片条目列表。
- 右栏显示当前选中卡片的竖向图片预览、复习状态、编辑和归档操作。
- 选中条目通过左侧强调线和浅色背景表达，不改变条目高度。
- 右栏预览保持 `1:2`，使用受限宽度避免挤出操作按钮。

### 空状态与搜索状态

- 空卡组显示单一录入操作，不注入示例卡片。
- 搜索无结果时保留搜索框，并显示清除搜索入口。
- 搜索只匹配文字 key；MVP 不提供标签和复杂筛选语法。

## 录入页

### 竖屏流程

竖屏录入严格分为两个二级页面。

第一步为 key 页面：

- 输入文字 key。
- 集中显示录入规则和画布说明。
- key 为空时，“进入绘图”必须给出字段错误和可感知反馈。

第二步为绘图页面：

- 顶栏显示当前 key 和返回入口。
- 第一行工具显示画笔、橡皮、移动和底图入口。
- 第二行固定显示颜色选项和紧凑笔刷大小滑块。
- 中央显示竖向 `1:2` 画布。
- 底部操作区固定显示撤销、重做和保存。
- 工具不能通过横向滚动隐藏在首屏之外。

### 横屏流程

横屏录入使用三部分协作：

| 区域 | 内容 |
| --- | --- |
| 中栏左侧 `64dp` 工具栏 | 画笔、橡皮、移动、颜色、竖向笔刷滑块、底图 |
| 中栏主体 | 竖向 `1:2` 绘图画布 |
| 右栏 | key、竖向预览、保存和清空 |

矮横屏下右栏预览缩小，保存与清空并排，确保两个操作都在首屏内。

### 画布契约

- 逻辑画布固定为 `1024×2048`，宽高比 `1:2`。
- 初始相机显示约 `35%` 的逻辑画布，用于后续移动和缩放。
- 画布边界必须清晰可见，笔迹不能越过实体画布边界。
- 底图使用 `FitCenter` 等比居中，禁止拉伸。
- 绘图预览和 PNG 导出必须调用同一个场景渲染器。
- 预览、卡片背面、列表缩略图和导出文件都使用相同方向，不做旋转补偿。
- 仅插入底图时允许保存；空白 value 不允许保存。
- 单点笔触必须保存为可见圆点，不能因路径点不足而丢失。

## 备份页

备份页在当前阶段只定义 MVP 界面：

- 导出全部数据。
- 从 ZIP 备份恢复。
- 显示最近一次备份文件名、时间和大小。
- 点击导入或导出后必须进入系统文件选择器或显示明确错误。

横屏右栏显示当前卡组数、卡片数和预计备份大小。竖屏通过信息弹层显示相同内容。

## 状态与反馈

### 必须跨旋转保存的状态

- 当前导航目的地。
- 当前卡组和当前学习卡片。
- 学习卡片当前正反面。
- 卡片页搜索词和选中条目。
- 录入步骤和文字 key。
- 笔迹、撤销栈、重做栈、底图和画布相机。
- 最近使用的画笔模式、颜色和大小。

### 用户反馈

- 保存、导入、导出、清空和归档必须显示成功、失败或取消结果。
- 字段校验错误显示在对应输入区域，同时可使用 Snackbar 汇总。
- 长任务显示进行中状态并禁用重复提交。
- 开发构建中的异常直接抛出并记录完整堆栈，不使用空 `catch` 或无提示失败。
- 发布构建将技术异常转换为中文用户提示，同时保留可诊断日志。

## 无障碍与触控笔

- 图标按钮必须提供中文内容描述和工具提示。
- 所有主要操作都需要触控可达，不能只依赖手势。
- 正文和控件文本满足可读对比度。
- 系统字体放大后，按钮文字允许换行，但不能与相邻控件重叠。
- 触控笔负责绘制，双指负责移动和缩放；橡皮端进入橡皮模式。
- 手掌误触和多点触控策略在绘图引擎规格中单独定义。

## 验收矩阵

必须在以下尺寸进行 Compose 截图测试和真机检查：

| 尺寸 | 预期布局 | 重点检查 |
| --- | --- | --- |
| `360×800` | 竖屏底部导航 | 竖卡占据主区域、绘图工具全部可见 |
| `800×360` | 横屏三栏 | key 不裁切、画布不遮挡、保存首屏可达 |
| `800×1280` | 竖屏底部导航 | 不因宽度较大误切换到三栏 |
| `1280×800` | 横屏三栏 | 列表、主工作区和右栏比例稳定 |

每个尺寸至少验证：

- 页面不存在横向溢出。
- 底部导航或导航栏不遮挡内容。
- 所有 value 图像保持 `1:2`，没有拉伸、裁切或方向错误。
- 文字不超出卡面、按钮和条目。
- 旋转前后的导航、选中和录入状态一致。
- 学习、卡片、录入和备份四个目的地均可进入和返回。

## 当前阶段不包含

- 卡片四元数姿态、拖拽分段函数和回弹动画。
- Room 实体、Repository 和旧数据迁移。
- 推荐权重和复习历史算法。
- SAF ZIP 的真实读写与事务导入。
- 云同步、账号系统和生产监控。
- 商店发布和生产级发布门禁。

这些能力分别在后续数据层、物理交互、业务 MVP 和发布阶段设计中定义。
