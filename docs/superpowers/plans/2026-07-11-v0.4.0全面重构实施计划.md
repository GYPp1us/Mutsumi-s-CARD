# v0.4.0 全面重构实施计划

> **供自动化开发代理使用：** 必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans`，按任务逐项实施。步骤使用复选框跟踪。

**目标：** 将现有占位式 Android MVP 重构为数据可靠、响应式布局完整、触控笔录入可用、学习卡片物理一致、卡组与备份具备真实交互的 `v0.4.0`，并发布可覆盖安装的签名 APK。

**架构：** 单 Activity + Jetpack Compose。Room 保存卡组、卡片和复习状态，DataStore 保存偏好，应用私有目录保存图片；Repository 通过 Flow 向分页面 ViewModel 暴露状态。学习卡片使用纯数学 `CardPose` 和单 Canvas 透视渲染器；绘图预览和 PNG 导出共享场景几何。

**技术栈：** Kotlin 2.2.21、Android Gradle Plugin 9.2.1、Gradle Wrapper 9.6.1、Jetpack Compose Material 3、Room、DataStore、Kotlin Coroutines、Android SAF、JUnit、Robolectric、Compose UI Test、GitHub Actions。

- [ ] **步骤 1：先写关键值失败测试**

验证：

- `distance=0` 返回起始姿态和原中心。
- `distance=screenWidth/4` 处内外函数值和一阶趋势连续。
- 超出邻域 `screenWidth/10` 时水平翻转达到 `180°`。
- 每次手势相对起始方向限制在 `0°..180°`。
- 背面起手与正面起手沿同一物理方向连续。
- 水平中心位移比例为 `0.075`。
- 竖直滚转上限为 `32°`。
- 上滑逐渐抑制横向位移。
- 回弹四元数 SLERP 走最短弧，不产生多圈旋转。

- [ ] **步骤 2：实现姿态模型**

```kotlin
data class CardPose(
    val center: Vec2,
    val orientation: Quaternion,
)

class StudyGestureMapper {
    fun poseForDrag(
        delta: Vec2,
        startPose: CardPose,
        startSide: CardSide,
        viewport: Size2,
    ): GestureProjection
}
```

邻域内角度为平滑倾斜；邻域外从边界角连续增加到最多 `180°`。中心位移和角度均由同一归一化距离与方向权重计算，不在渲染层判断手势动作。

- [ ] **步骤 3：实现回弹与中断续接**

中心使用三次非线性插值，朝向使用最短弧 SLERP。回弹期间再次触摸，以动画当前 `CardPose` 为新起点，不回到 committed 静止姿态。

- [ ] **步骤 4：实现单 Canvas 物理渲染**

渲染器公开接口只接收卡片纹理和姿态：

```kotlin
fun DrawScope.drawPhysicalCard(
    front: ImageBitmap,
    back: ImageBitmap,
    pose: CardPose,
    cardSize: Size2,
)
```

通过四元数旋转四角、透视投影和 `Matrix.setPolyToPoly` 绘制当前朝向的固定纹理。正背面不使用 Compose 内容树切换、alpha 虚影、阴影或双层 `graphicsLayer`。
