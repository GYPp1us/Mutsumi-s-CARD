# v0.5.0 双面卡片与 Markdown 绘图设计

## 目标

卡片的正反面都可以是最终 PNG 图片。背面必须有内容，正面可以为空；正面为空时，学习页显示文字 `key`。一次录入同时编辑两个卡面，保存后只持久化两张成品 PNG 和文字 key，不保存笔迹、底图或 Markdown 的可编辑工程。

## 数据与兼容

- `CardEntity` 和领域 `MemoryCard` 增加可空 `frontImagePath`；既有 `valueImagePath` 始终代表背面。
- Room 数据库升级到版本 2，通过 `MIGRATION_1_2` 对 `cards` 增加可空 `frontImagePath` 列。旧卡片自动作为“无正面图片”的卡片读取。
- 新建时 Repository 先写可选正面 PNG，再写必需背面 PNG；数据库写入失败必须删除本次新写入的全部图片，并继续抛出原异常。
- 本地 ZIP 和 WebDAV 快照都包含正面与背面引用。旧备份 JSON 因 `frontImagePath` 默认值为 `null` 仍可导入。
- 云端 `+N/-N` 指纹含卡组、key、归档状态、正反面 PNG 哈希；复习权重和全部复习状态不计入增量，但仍写入快照、可恢复。

## 录入布局

- 录入页面进入即请求横屏；若设备或系统拒绝横屏，页面只显示横屏阻断提示，不显示错位的编辑器。
- 横屏主布局为一行：左侧固定约 `208dp` 的共享工具栏，右侧正面、背面两张并行银行卡比例画布。页面外围最小内边距 `6dp`，两栏间距 `8dp`。
- 每张画布使用 `1024 x 1624` 的 ISO ID-1 竖向比例，显示尺寸在其所在栏内最大化且保持比例。点击或在任一画布上开始手势会选中该面；工具只对选中面生效。
- 工具栏依次提供：文字 key、画笔/橡皮/移动/Markdown、颜色、笔刷尺寸、插入底图、Markdown 编辑或预览、撤销、清空、保存。正面可空，背面必须包含笔迹、底图或非空 Markdown。
- 保存成功后清空两个草稿，选中正面并进入学习页；保存失败由上层协程抛出，不得伪造成功反馈。

## 图层和相机

每个卡面独立维护草稿，图层顺序固定如下：

1. 白色卡面背景。
2. 底图：位于无限世界坐标，跟随该卡面的相机平移、缩放，始终等比显示，不裁切、不拉伸。
3. Markdown：透明背景，固定在 `1024 x 1624` 的成品卡面坐标，不随相机平移或缩放。
4. 笔迹：位于无限世界坐标，和底图共用相机。

相机只保存中心、缩放和视口。`CanvasCamera.transform` 以当前帧双指质心为锚点，将手势平移和缩放同时转换为新的世界中心；不对平移或缩放设置边界。视口尺寸改变时只更新视口，不重置中心或缩放。

导出时使用当前相机可见世界区域重建到 `1024 x 1624`，依照同一图层顺序栅格化。这样保存 PNG 与当前预览的卡面视口一致；Markdown 始终按照成品卡面坐标渲染。

## 笔迹输入

- 禁止使用 `detectDragGestures` 作为画笔入口，因为其 touch slop 会吞掉短笔画。
- 使用 `awaitEachGesture` 与 `awaitFirstDown(requireUnconsumed = false)`：手指或触控笔落下时立即写入第一个世界坐标点；每个后续 PointerEvent 直接采样，不要求越过阈值；抬起时即使只有一个点也保存为圆点笔迹。
- 笔宽在落笔时换算为世界单位，预览和导出时乘各自相机缩放，因此缩放画布不会改变已画笔迹的视觉宽度。
- 橡皮擦同样按下立即命中；移动工具只更新相机，不会移动 Markdown 固定层。

## Markdown

- Markdown 是整面文档层，透明、不可拖拽、不可缩放；源码编辑使用等宽字体。
- 默认显示预览。点击工具栏切换按钮，或在当前选中卡面按 `Ctrl+W`，在源码与预览之间切换。
- 不做实时预览：仅在进入预览或导出时通过 Markwon 渲染。
- 支持简单 Markdown、GFM 表格和 `$...$`/`$$...$$` LaTeX；不支持 HTML、脚本、网络图片或自定义 CSS。

## 学习页

- 卡片是单一刚体。正面存在 `frontImagePath` 时显示正面 PNG，否则显示 key；背面显示 `valueImagePath`。
- 横屏/竖屏的手势邻域半径和横向翻转带都以学习区域短边 `min(width, height)` 计算，防止横屏宽度放大导致操作距离失真。纵向离场目标仍以真实高度计算。
- 正反面渲染基于同一个姿态，仅以 alpha 和镜像切换可见内容；禁止通过替换整棵内容树实现翻面。

## 验收

1. 正面为空、背面有内容时可保存并在学习页显示 key；正反面都有内容时学习页能正确显示两张 PNG。
2. 横屏录入两个卡面同步可用，正反面独立拥有底图、笔迹、相机和 Markdown。
3. 小于一个触控 slop 的落笔也能保存为可见圆点或短笔迹。
4. 平移与缩放没有边界，双指缩放围绕实时双指质心。
5. ZIP/WebDAV 导出、导入、云端增量都不会遗漏正面 PNG；只改变复习状态不会增加 `+N`。
